First-time Setup (5 min, once only)
  1. Install Node.js from nodejs.org — download the LTS version
  2. Put server.js and package.json into a folder called wa-bridge
  3. Open Terminal / Command Prompt, navigate to that folder:
    cd path/to/wa-bridge
  4. Run once to install packages:
    npm install
  5. Start the bridge:
    npm start
  6. Come back here, click Connect — scan the QR that appears
  7. Done ✓ — session saved, future starts need no QR scan
💡 Each session: open Terminal → cd wa-bridge → npm start → click Connect here → scan chats → Ctrl+C when done.

Google Sheets Connection
How to Connect (Step-by-Step)
  1. Open Google Sheets → create a new blank spreadsheet named Masala Book
  2. In the menu go to Extensions → Apps Script
  3. Delete the default code, paste the entire contents of the Code.gs file provided
  4. Click Save (Ctrl+S / Cmd+S)
  5. Click Deploy → New Deployment
  6. Type: Web app · Execute as: Me · Who has access: Anyone
  7. Click Deploy, authorise when prompted, then copy the Web app URL
  8. Paste that URL in the field above and click Test Connection
  9. On first connection, sheets will be auto-created with correct headers
About the Apps Script (Code.gs)

The Apps Script must handle these actions: getData, saveOrder, markDelivered, deleteOrder, saveMasala, deleteMasala, saveRecipeRows (batch — receives an array), deleteAllRecipeRows (batch delete by masala name), saveRawMaterial, deleteRawMaterial, ping.

⚠️ If pending orders always show as "Delivered", check that your Code.gs stores and returns the OrderType column correctly in the Orders sheet.
